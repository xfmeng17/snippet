# 模式 B 模板：课程讲解

## README.md 模板

```markdown
# {主题} 深度学习课程 —— 基于 {项目名} 源码

## 课程目标
一句话说明学完能掌握什么

## 前置知识
列出需要的基础知识（语言、数据结构等）

## 课程大纲
| 课时 | 文件 | 主题 | 预计时间 | 难度 |
|------|------|------|----------|------|
| 01 | [01_xxx.md](01_xxx.md) | 直觉建立 | 30min | 基础 |
| 02 | [02_xxx.md](02_xxx.md) | 核心数据结构 | 45min | 基础 |
| ... | ... | ... | ... | ... |

**总计约 N 小时**

## 阅读建议
1. 先读课时 01，建立直觉，不看代码
2. 课时 02-04 配合源码一起读
3. ...

## 源码导航
关键目录和文件的树形结构 + 简要说明
```

## N.2 {第二个知识点}

## N.X 关键函数索引（如有）
| 函数 | 位置 | 作用 |

## 课时设计原则

### 推荐的课时编排顺序

1. **直觉课**（1 课）：用日常类比解释核心思想，零代码
2. **数据结构课**（1-2 课）：内存布局、核心数据结构
3. **核心算法课**（2-3 课）：主要算法的详细讲解
4. **端到端课**（1 课）：追踪一个完整操作从入口到结果
5. **工程细节课**（1-2 课）：并发、序列化、性能优化等
6. **进阶/对比课**（1-2 课）：变体实现、设计取舍
7. **总结课**（1 课）：全局对比、源码阅读路线图

### 可视化要求

每课至少包含以下之一：
- ASCII art 架构/流程图
- 步骤可视化（如搜索过程的逐步演示）
- 对比表格
- 内存布局图

### 源码准确性要求

- **禁止凭印象编写代码片段**。写课时前必须用 Read 工具读取实际源码，确认：
  - 成员变量名和类型（不能写错名字）
  - 函数签名和参数列表
  - 行号（标注后再用 Read 验证）
- **伪代码与真实代码明确区分**：伪代码标注"(简化)"，真实代码标注文件路径和行号
- **数值计算写出完整公式**：涉及字节数、参数范围时，从源码找到计算公式，代入具体值，不要直接写结果

### 封装层/胶水层课时指南

封装层（wrapper、adapter、bridge）不含核心算法，容易写成"API 罗列"。应聚焦于：
1. **为什么需要这层**：用对照表展示两侧（算法层 vs 用户需求）的 gap
2. **关键设计模式**：异常边界转换、线程安全策略、状态机、参数校验规则的意图
3. **非平凡逻辑**：不是简单转发的部分（如 Tombstone 恢复、连通性检查、能力声明）
4. 如果信息量不足以独立成课，合并到相邻课时

### 深度均匀性检查

生成全部课时后，回顾检查：
- 相邻课时的信息密度是否差距过大（不应出现某课 400 行、某课 100 行的情况）
- 对比类内容（如 A vs B）双方是否达到相近的讲解深度
- 如果某个主题确实只需概览，在 README 中明确标注为"概览"级别
