# 模式 A 模板：源码笔记

## README.md 模板

```markdown
# {项目名}：{主题} 源码笔记

## 概述
一段话说明这个主题涵盖什么

## 文件导航
| 文件 | 内容 | 难度 |
|------|------|------|
| [topic1.md](topic1.md) | ... | 基础 |
| [topic2.md](topic2.md) | ... | 进阶 |

## 源码关键路径
简要列出涉及的源码目录和文件
```

## 笔记文件模板

```markdown
# {主题名}

> 对应源码：`{关键文件路径}`

## 概述
这个模块/组件是什么，解决什么问题（2-3 句话）

## 架构
用 ASCII art 画出结构关系、继承层次或数据流

## 核心实现
提取关键代码片段，标注 `文件:行号`，加中文注释
不要大段 copy，只保留关键逻辑

## 端到端追踪（推荐）
选一个典型操作（如一次查询、一次插入），从入口追踪到结果

## 对比（如有）
相似概念/实现用表格对比

## 配置参数（如有）
| 参数 | 类型 | 默认值 | 含义 |
|------|------|--------|------|

## 小结
3-5 行总结要点，用代码块或列表
```
