# 模式 E 模板: 教学互动（quiz / explain）

> 基于 Active Recall（主动回忆）和 Feynman Technique（费曼技巧）的教学互动模板。

## 子模式一: quiz（主动回忆 + 苏格拉底追问）

### 触发方式
- `/learn-vec quiz {topic}` — 针对特定主题的回忆测试
- 课程模式中每课末尾自动附带 3-5 道思考题

### 出题原则

1. **不考记忆，考理解**: 避免"XX 的定义是什么"，改为"如果去掉 XX 会怎样"
2. **苏格拉底式**: 不直接给答案，用引导问题帮用户自己发现
3. **递进深度**: 从概念辨析 → 参数推理 → 设计决策 → 代码补全

### 题型模板

**概念辨析（L1-L2）**:
> HNSW 的 Level 0 层和上层（Level 1+）在结构上有什么区别？如果去掉上层只保留 Level 0，搜索性能会怎样变化？

**参数推理（L2）**:
> 你有 100 万个 128 维 float32 向量，要求内存不超过 500MB。PQ 的 m 和 nbits 参数应该怎么选？请写出内存计算过程。

**设计决策（L2-L3）**:
> HNSW 在选择新节点的邻居时，有 simple 和 heuristic 两种策略。heuristic 策略额外考虑了什么？在什么数据分布下差异最明显？

**代码补全（L2-L3）**:
> 以下是 HNSW 搜索的简化伪代码，`???` 处应该填什么？为什么？
> ```
> while candidates is not empty:
>     c = candidates.pop_nearest()
>     if distance(c, query) > ???:
>         break
>     for neighbor in get_neighbors(c):
>         ...
> ```

### 回答评估

用户回答后，不直接判对错，而是:
1. **肯定正确部分**: "你提到了 XX，这是对的"
2. **追问不完整部分**: "那如果数据是聚簇分布的呢？"
3. **引导错误部分**: "你说 M 越大越好，但想想看，M=1000 时搜索一个节点的邻居需要多少次距离计算？"
4. 3 轮追问后给出完整答案和解析

### quiz 输出格式

```markdown
# 主题回忆测试: {topic}

## 题目 1/N
{题目内容}

### 你的回答
{用户回答}

### 追问
{苏格拉底式追问}

### 解析
{完整解析，标注源码位置}
```

## 子模式二: explain（费曼讲解）

### 触发方式
- `/learn-vec explain {concept}` — 用户向 AI 解释一个概念

### 流程

1. **用户开始解释**: AI 扮演一个聪明但完全不懂向量检索的小白
2. **AI 追问（3-5 轮）**: 针对用户解释中模糊或遗漏的部分提问
3. **AI 给出评估**: 清晰度/准确性/深度 + 发现的知识盲点 + 复习建议

### AI 追问原则

- **抓模糊词**: "你说'差不多'，具体差多少？"  "你说'很快'，大概什么量级？"
- **抓因果链断裂**: "你说 A 导致 B，但中间的过程是什么？"
- **抓隐含假设**: "你说的情况在向量归一化后还成立吗？"
- **不追问琐碎细节**: 只关注核心理解是否到位

### 追问示例

用户: "HNSW 就是把向量放到一个多层的图里，搜索的时候从上往下找"
AI: "多层的图听起来像金字塔。上面几层和最下面一层有什么不同？每层有多少个节点？"

用户: "上面的层节点少，下面的多，这样搜索快"
AI: "为什么节点少搜索就快？如果我随机删掉一半节点也能加速吗？上层的节点是怎么选出来的？"

### 评估维度

| 维度 | 评分 | 说明 |
|------|------|------|
| **清晰度** | ⭐1-5 | 小白能否听懂？类比是否恰当？ |
| **准确性** | ⭐1-5 | 有无事实错误或误导性简化？ |
| **深度** | ⭐1-5 | 是否触及 "Why" 层面？还是只停在 "What"？ |

### explain 输出格式

```markdown
# 费曼讲解评估: {concept}

## 对话记录
{用户解释 + AI 追问的完整对话}

## 评估
| 维度 | 评分 | 说明 |
|------|------|------|
| 清晰度 | ⭐N | ... |
| 准确性 | ⭐N | ... |
| 深度   | ⭐N | ... |

## 发现的知识盲点
- {盲点 1}: {表现 + 建议复习的课时/资料}
- {盲点 2}: ...

## 复习建议
{针对盲点的具体学习建议}
```
