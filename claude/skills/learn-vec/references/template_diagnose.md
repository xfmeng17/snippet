# 模式 D 模板: 水平诊断

> 5 题快速诊断用户向量检索知识水平，结果联动 course 模式的课时跳过策略。

## Level 定义

| Level | 名称 | 特征 |
|-------|------|------|
| **L0** | 零基础 | 不知道 ANN，不了解距离度量，可能只知道"向量"是个数学概念 |
| **L1** | 基础 | 知道 L2/Cosine 距离，听说过 HNSW/IVF，但不清楚内部原理 |
| **L2** | 中级 | 理解至少一种 ANN 算法的核心原理，能解释 recall vs QPS 的 trade-off |
| **L3** | 高级 | 掌握多种算法的设计取舍，有参数调优和 benchmark 经验 |

## 诊断流程

1. 向用户提出 5 道诊断题（可选择回答或跳过）
2. 根据回答质量判定 Level
3. 输出诊断结果 + 推荐学习路径
4. 如果后续进入 course 模式，自动应用跳过策略

## 5 道诊断题

### Q1: 概念题（判定 L0 vs L1）
> 向量检索和传统数据库的关键词查询有什么本质区别？

**L0 答案特征**: 不清楚区别，或只能说"一个用向量一个用关键词"
**L1+ 答案特征**: 提到"相似度/距离"、"语义匹配"、"近似最近邻"等概念

### Q2: 度量题（判定 L1 细化）
> L2 距离和 Cosine 相似度分别适合什么场景？它们有没有等价的情况？

**L1 答案特征**: 知道两者区别但不清楚等价条件
**L2+ 答案特征**: 知道归一化后 L2 排序等价于 Cosine 排序

### Q3: 算法题（判定 L1 vs L2）
> 请描述你了解的任意一种 ANN 算法的核心思想（如 HNSW、IVF、LSH 等）。

**L1 答案特征**: 只能说名字和大概用途
**L2 答案特征**: 能描述核心数据结构和搜索策略
**L3 答案特征**: 能深入到实现细节和 trade-off

### Q4: 工程题（判定 L2 vs L3）
> HNSW 的参数 M 和 ef_construction 分别影响什么？增大它们有什么代价？

**L2 答案特征**: 知道大致影响方向但不精确
**L3 答案特征**: 能说清 M 影响图密度/内存/搜索时间，ef 影响构建质量/速度的 trade-off

### Q5: 实践题（判定 L3）
> 你的向量检索系统 recall@10=0.80，用户希望达到 0.95。你会如何排查和优化？

**L2 答案特征**: 只能说"调大参数"
**L3 答案特征**: 能从数据分布、距离选择、索引参数、查询参数、量化精度等多角度分析

## 判定规则

```
Q1 未答出 → L0
Q1 答出 + Q3 未答出 → L1
Q3 答出 + Q4 未答出 → L2
Q4+Q5 答出 → L3
```

## 与 course 模式联动

诊断结果自动影响课程生成策略:

| Level | 课程调整 |
|-------|---------|
| **L0** | 从第 1 课开始，直觉课增加更多日常类比，距离计算课从零讲起 |
| **L1** | 跳过纯直觉课（合并为 0.5 课的回顾），从数据结构课开始深入 |
| **L2** | 跳过前 3 课（直觉+距离+暴力搜索），从核心算法深度课开始 |
| **L3** | 只生成进阶课时: 工程优化、参数调优、跨算法对比、最新进展（RaBitQ 等）|
