## README.md 模板

```markdown
# {项目/库名}: {主题} 源码笔记

## 概述
一段话说明这个主题涵盖什么，学完能理解什么。

## 文件导航
| 文件 | 内容 | 难度 |
|------|------|------|
| [distance.md](distance.md) | 距离计算 + SIMD 加速 | 基础 |
| [index_structure.md](index_structure.md) | 索引数据结构 | 中级 |
| ... | ... | ... |

## 源码关键路径
关键目录和文件的树形结构 + 简要说明

## 阅读建议
推荐阅读顺序 + 每篇的前置依赖
```

## 笔记文件模板

```markdown
# {主题名}

> 对应源码: `{关键文件路径}`

## 概述
这个模块/组件是什么，解决什么问题（2-3 句话）

## 架构
用 ASCII art 画出结构关系、继承层次或数据流

## 核心实现
提取关键代码片段，标注 `文件:行号`，加中文注释。
不要大段 copy，只保留关键逻辑。

## 端到端追踪（推荐）
选一个典型操作（如一次 search），从 API 入口追踪到返回结果

## 对比（如有）
相似概念/实现用表格对比

## 小结
3-5 行总结要点
```

## 向量库笔记推荐组织

按以下维度组织笔记，每个维度一个 `.md` 文件:

| 维度 | 文件名建议 | 重点内容 |
|------|-----------|---------|
| **距离计算** | `distance.md` | L2/IP/Cosine 公式、SIMD 版本（SSE/AVX/AVX-512）、热路径分析 |
| **索引结构** | `index_structure.md` | 核心数据结构、内存布局（画字节偏移图）、关键常量 |
| **查询引擎** | `search_engine.md` | 搜索主循环、候选列表管理、beam search/greedy 策略 |
| **构建流程** | `build_process.md` | 索引构建算法、批量/增量构建、并行策略 |
| **序列化** | `serialization.md` | 存储格式、内存映射（mmap）、版本兼容 |
| **工程优化** | `engineering.md` | prefetch、内存对齐、缓存友好、多线程策略 |

### 向量领域特化要求

每篇笔记在通用模板基础上，额外关注:

- **距离计算热路径**: 标注该模块涉及的距离计算函数及 SIMD 版本
- **内存布局字节图**: 核心数据结构画出每个字段的字节偏移
- **SIMD 版本说明**: 标注 SSE4.2 / AVX2 / AVX-512 / NEON 的支持情况
- **量化参数**: 涉及量化时标注 bits/子空间数/码本大小等关键参数
