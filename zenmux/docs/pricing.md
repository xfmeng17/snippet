# ZenMux 收费方式详解

> 信息来源: [ZenMux 订阅制文档](https://zenmux.ai/docs/zh/guide/subscription.html)、[PAYG 文档](https://zenmux.ai/docs/zh/guide/pay-as-you-go.html)、[计费透明度](https://zenmux.ai/docs/zh/guide/observability/pricing.html)
>
> 更新日期: 2026-03-26

ZenMux 是 LLM API 聚合平台，一个账号、一个 API Key 即可调用 200+ 模型（Anthropic、OpenAI、Google、xAI 等）。计费方式分为**订阅制**和 **PAYG（按量付费）**两种，可灵活切换和互补。

---

## 一、订阅制（Builder Plan）

### 核心概念: Flow

Flow 是 ZenMux 订阅制**专用**的计费单位（PAYG 不用 Flow，直接按美元扣费）。

**一句话理解**: Flow 就是"订阅额度的通用货币"。你调不同模型消耗的 Flow 不同，但都从同一个 Flow 池子里扣。

**汇率**: 1 Flow = $0.02525（约 **40 Flows ≈ $1**）

> 该汇率可能根据市场状况和模型定价变化周期性调整，最新汇率在 Pricing 页面实时公布。

#### Flow 的换算逻辑

Flow 是一个复合计费单位，包含两部分:
- **token 消耗**: 按模型官方 token 价格折算
- **调用开销**: 每次 API 请求的固定开销

换算公式（简化，忽略调用开销）:

```
消耗的 Flow 数 ≈ (输入 tokens × 输入单价 + 输出 tokens × 输出单价) ÷ $0.02525
```

#### 1 Flow 能调多少 Claude tokens？

以 1 Flow = $0.02525 为基准，**纯输入**和**纯输出**的理论上限:

| 模型 | 官方输入价 | 1 Flow 纯输入 | 官方输出价 | 1 Flow 纯输出 |
|------|-----------|--------------|-----------|--------------|
| Claude Haiku 4.5 | $1/1M | **25,250 tokens** | $5/1M | **5,050 tokens** |
| Claude Sonnet 4.6 | $3/1M | **8,417 tokens** | $15/1M | **1,683 tokens** |
| Claude Opus 4.6 | $5/1M | **5,050 tokens** | $25/1M | **1,010 tokens** |

> 计算方式: 1 Flow 纯输入 = $0.02525 ÷ (输入单价 / 1M) = $0.02525 × 1M ÷ 输入单价

#### 实际场景估算: Claude Code 一次对话消耗多少 Flow？

Claude Code 的典型请求模式是**大输入（代码上下文）+ 中等输出（回复/改动）**。
假设一次典型交互: 输入 10K tokens + 输出 2K tokens。

| 模型 | 输入成本 | 输出成本 | 总成本 | 消耗 Flow |
|------|---------|---------|--------|----------|
| Haiku 4.5 | $0.01 | $0.01 | $0.02 | **~0.8** |
| Sonnet 4.6 | $0.03 | $0.03 | $0.06 | **~2.4** |
| Opus 4.6 | $0.05 | $0.05 | $0.10 | **~4.0** |

所以 Pro 套餐的 50 Flows/5h，大约能支撑:
- **Haiku 4.5**: ~62 次对话
- **Sonnet 4.6**: ~21 次对话
- **Opus 4.6**: ~12 次对话

> 这是"每次 10K 输入 + 2K 输出"的粗略估算。Claude Code 实际使用中，随着上下文增长，后续交互的输入 token 会越来越大（可能到 50K-100K），单次消耗会显著增加。

### 套餐一览

| 套餐 | 月费 | 5h 配额 | 周上限 | 月上限 | 月上限等值 USD | 价格杠杆 |
|------|------|---------|--------|--------|---------------|---------|
| Free | $0 | 5 Flows | 50.4 | 216 | $5.45 | — |
| **Pro** | **$20** | **50 Flows** | **504** | **2,160** | **$54.54** | **~2.7x** |
| **Max** | **$100** | **300 Flows** | **3,024** | **12,960** | **$327.24** | **~3.3x** |
| **Ultra** | **$200** | **800 Flows** | **8,064** | **34,560** | **$872.64** | **~4.4x** |

> 价格杠杆 = 月上限等值 USD ÷ 月费。意味着你花 $200 订阅 Ultra，最多可获得 ~$872 等值的 API 调用量。

### 配额重置机制

- **5 小时窗口**: 从第一次请求开始计量，每 5 小时重置一次
- **周限额**: 从第一次请求开始计量，每满 7 天重置一次
- 两个窗口独立运作，**取先触达上限者**

### 各套餐 5 小时能用多少？

以"每次 10K 输入 + 2K 输出"的典型 Claude Code 交互为例:

| 套餐 | 5h 配额 | 等值 USD | Haiku 4.5 次数 | Sonnet 4.6 次数 | Opus 4.6 次数 |
|------|---------|---------|---------------|----------------|--------------|
| Free | 5 Flows | $0.13 | ~6 次 | ~2 次 | ~1 次 |
| Pro | 50 Flows | $1.26 | ~62 次 | ~21 次 | ~12 次 |
| Max | 300 Flows | $7.58 | ~375 次 | ~125 次 | ~75 次 |
| Ultra | 800 Flows | $20.20 | ~1000 次 | ~333 次 | ~200 次 |

> 随着对话变长、上下文膨胀，单次消耗 Flow 会增加。重度 Claude Code 用户（Opus，长上下文）建议 Max 或 Ultra。

### 订阅溢出机制

启用 Extra Usage 后：
1. 订阅配额用尽 → 自动从 PAYG 余额扣费
2. 订阅配额窗口重置 → 自动切回订阅制计费
3. 全程无需手动切换 API Key

## 二、PAYG（Pay As You Go 按量付费）

### 核心特点

- **预充值 + 按实际消耗扣费**，1 积分 = $1 USD 等值 API 调用
- **定价倍率: 1:1 对标官方原价**（不加价）
- **充值赠送 20%**: 充 $100 得 $120 积分，充 $500 得 $600，充 $1000 得 $1200
- 无 RPM 限制、无并发限制
- 生产级 SLA 保证

### Claude 模型 PAYG 价格（= Anthropic 官方价）

| 模型 | 输入 (per 1M tokens) | 输出 (per 1M tokens) |
|------|---------------------|---------------------|
| Claude Haiku 4.5 | $1 | $5 |
| Claude Sonnet 4.6 | $3 | $15 |
| Claude Opus 4.6 | $5 | $25 |

> PAYG 倍率为 1:1，即 ZenMux 上的价格与直接调 Anthropic API 相同。再加上充值赠送 20%，实际成本约为官方的 **83%**（1/1.2）。

### 长上下文计费（>200K tokens）

当请求 token 超过 200K 时，超出部分按如下倍率计费:

| 部分 | 输入倍率 | 输出倍率 |
|------|---------|---------|
| ≤200K tokens | 1x（标准价） | 1x（标准价） |
| >200K tokens | **2x** | **1.5x** |

适用模型: Claude Opus 4.6、Claude Sonnet 4.5、Claude Sonnet 4（均支持 1M context）。

配合 prompt caching 可降低长上下文输入成本最高 90%。

## 三、订阅 vs PAYG 怎么选？

| 维度 | 订阅制 | PAYG |
|------|--------|------|
| 适用场景 | 个人开发、学习、Vibe Coding | 生产环境、商业项目 |
| 成本优势 | 3~4 倍杠杆（用满的话） | 1:1 官方价 + 充值送 20% |
| 速率限制 | 有配额上限（5h/周） | 无限制 |
| 最低门槛 | $0（Free）/ $20（Pro） | 按需充值 |
| 能否混用 | 可以，配额用完自动溢出到 PAYG | 可以，随时切换 |

### $100 花在三种方式上的对比

| 方式 | 花 $100 | 实际可用 API 价值 | 倍率 |
|------|---------|-----------------|------|
| Anthropic 直连 | $100 充值 | $100 | 1.0x |
| ZenMux PAYG | $100 充值 | **$120**（送 20%） | 1.2x |
| ZenMux Max 订阅 | $100/月 | **$327**/月（用满） | 3.3x |

### 订阅制为什么能给 3~4 倍杠杆？

本质上和**健身房会员卡**一个道理——**统计套利**:

1. **ZenMux 赌的是"用不满"**。Max 套餐每 5 小时给 300 Flows，你得每个窗口都用满才能拿到 $327 的月上限。一天 4.8 个窗口，一个月 ~144 个窗口——你睡觉、开会、不写代码的时候，窗口就白过了。ZenMux 统计上知道大多数人实际利用率远低于 100%。

2. **滚动窗口是限流器**。不让你攒着一口气用完，而是把用量打散到时间轴上。ZenMux 的上游 API 成本因此是平滑的，峰值可控。

3. **越贵的套餐杠杆越大**。Ultra $200 给了 4.4x 杠杆，但你要每 5 小时消耗 800 Flows 才能用满——那是非常重度的 7×24 使用，现实中极少有人做到。

**一句话总结**: 订阅省钱的前提是**你用得够多**。用得越满，省得越多；用不满，就是在补贴别人。

### $100 能买多少 tokens？

以 5:1 输入输出比（Claude Code 典型场景）估算:

| 模型 | 混合单价 (per 1M) | $100 总 tokens | 每次 15K tokens 约几次交互 | 每天 40 次约用几天 |
|------|-------------------|---------------|-------------------------|------------------|
| Haiku 4.5 | $1.67 | **~60M** | ~4,000 次 | ~100 天 |
| Sonnet 4.6 | $5.00 | **~20M** | ~1,333 次 | ~33 天 |
| Opus 4.6 | $8.33 | **~12M** | ~800 次 | ~20 天 |

> 混合单价 = (5 × 输入价 + 1 × 输出价) ÷ 6。"15K tokens/次"= 10K 输入 + 2K 输出 + 3K thinking。

### 个人开发者建议

- 轻度使用（每天几次对话）: **Free** 够用
- 日常 Claude Code 开发: **Pro $20** 性价比最高，5h/50 Flows 对轻中度编码足够
- 重度使用（全天 Claude Code）: **Max $100** 或 **Ultra $200**，配合 PAYG 兜底

## 四、附录: Anthropic 官方 Claude API 怎么收费？

> 信息来源: [Anthropic API Pricing](https://www.nops.io/blog/anthropic-api-pricing/)、[IntuitionLabs Claude Pricing](https://intuitionlabs.ai/articles/claude-pricing-plans-api-costs)

### 计费模型: 按 token，不是按请求次数

**Claude API 按 token 计费，不是按 QPS（query-per-second）或请求次数。** 没有 per-request 固定费用。

- **输入 tokens**: 你发过去的所有内容（prompt、system message、聊天历史、文档、工具定义等）
- **输出 tokens**: Claude 返回的所有内容（回复文本、工具调用参数、JSON 等）
- **thinking tokens**: extended thinking 的内部推理过程，**按输出 token 价格计费**，不显示在回复中但花钱

费用公式:
```
费用 = (输入 tokens / 1M × 输入单价) + (输出 tokens / 1M × 输出单价)
```

同一次请求，30 tokens 和 30,000 tokens 的费用差 1000 倍——所以**不是按次收费**。

### 当前价格（2026 年 3 月）

| 模型 | 输入 (per 1M) | 输出 (per 1M) | 定位 |
|------|-------------|-------------|------|
| Claude Haiku 4.5 | $1 | $5 | 最快、最便宜 |
| Claude Sonnet 4.6 | $3 | $15 | 均衡之选 |
| Claude Opus 4.6 | $5 | $25 | 最强能力 |

### 两大省钱机制

**1. Prompt Caching（提示缓存）**

| 操作 | 相对标准价 | 说明 |
|------|-----------|------|
| 缓存写入 | 1.25x | 首次缓存，略贵 |
| 缓存命中 | **0.1x** | 后续命中，**省 90%** |

- TTL 5 分钟（5 分钟内重复使用同一 prompt 即可命中）
- 适合: 长 system prompt、反复分析同一份文档

**2. Batch API（批处理）**

| 模型 | 标准输入 → Batch 输入 | 标准输出 → Batch 输出 |
|------|---------------------|---------------------|
| Sonnet 4.6 | $3 → **$1.50** | $15 → **$7.50** |
| Opus 4.6 | $5 → **$2.50** | $25 → **$12.50** |

- **半价**，24 小时内完成，适合非实时批量任务

### 速率限制（Rate Limits）

Anthropic 对 API 有三层限制:
- **RPM**: 每分钟请求数
- **TPM**: 每分钟 token 数
- **TPD**: 每天 token 数

限额随使用层级（Tier）提升。可预充值提高额度。

### Anthropic 原生订阅（Pro/Max）

> 信息来源: [Claude Max Plan](https://intuitionlabs.ai/articles/claude-max-plan-pricing-usage-limits)、[Claude Code Pricing](https://www.ssdnodes.com/blog/claude-code-pricing-in-2026-every-plan-explained-pro-max-api-teams/)

Anthropic 自己也有订阅制，用于 claude.ai 网页端 + Claude Desktop + Claude Code:

| 套餐 | 月费 | 用量倍率 | 含 Claude Code |
|------|------|---------|---------------|
| Free | $0 | 1x（~20 条/天） | 不含 |
| Pro | $20 | 5x Free | 含 |
| Max 5x | $100 | 5x Pro（= 25x Free） | 含 |
| Max 20x | $200 | 20x Pro（= 100x Free） | 含 |

**限流方式和 ZenMux 完全一样**: 5 小时滚动窗口 + 7 天周上限。

#### 是按 QPS 限制还是按 token 限制？

**按 token 总量限制，不是按 QPS。** 没有"每秒最多几个请求"的限制。

- 你可以一秒内连发 10 条消息，不会被 QPS 拦截
- 但每条消息消耗的 token 数不同——一条短回复几百 tokens，一条带大量代码上下文的请求可能几万 tokens
- 5 小时窗口限制的是 **token 消耗总量**，不是消息条数
- Anthropic 故意不公布具体数字，但社区经验:
  - Pro: ~10-40 次 Claude Code 交互/5h
  - Max 5x: ~50-200 次/5h
  - Max 20x: ~200-800 次/5h

> 这就是"削峰填谷"——5 小时窗口不让你攒着一口气烧完月配额，把消耗平摊到时间轴上。Anthropic 和 ZenMux 用的是同一套思路。

#### Extra Usage（溢出）

和 ZenMux 一样，Anthropic 订阅也支持溢出: 配额用完后按 API 标准价继续扣费，可设置月度消费上限。

#### API vs 订阅是两套体系

- **Claude Pro/Max 订阅**（$20/$100/$200/月）: Web/桌面端 + Claude Code，token 消耗从订阅配额扣
- **API 用量**: 单独计费，预充值 credits，按 token 扣费，适合集成到自己的应用
- 两者完全独立，互不影响

### ZenMux PAYG 与 Anthropic 直连对比

| 维度 | Anthropic 直连 | ZenMux PAYG |
|------|---------------|-------------|
| 单价 | 官方价 | 官方价（1:1） |
| 充值优惠 | 无 | **充值送 20%** |
| 实际成本 | 100% | **~83%** |
| 多模型支持 | 仅 Claude | 200+ 模型统一 Key |
| 速率限制 | 有 Tier 限制 | 无限制 |
| 故障保障 | 无 | LLM Insurance 赔付 |
